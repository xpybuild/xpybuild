  3  # This module holds definitions for various PathSet classes. A path set is a  
  4  # lazily-resolved, thread-safe set of file/directory absolute paths, with  
  5  # an optional (perhaps trivial) mapping to relative 'destination' paths  
  6  # (which is used by some targets, e.g. copy). Where possible, ordering of  
  7  # the items in a compound pathset is preserved.  
  8  # 
  9  # A PathSet's contents are not resolved until after the  
 10  # initialization phase of the build is complete, since resolution may involve  
 11  # expensive file system operations.  
 12  # 
 13  # When using a pathset to specify paths located under a directory that  
 14  # is generated as part of the build process, always use  
 15  # DirGeneratedByTarget(dirpath) to wrap the reference to the generated  
 16  # directory, which forces the evaluation of its contents to be delayed  
 17  # until the target dependency has actually been built.
